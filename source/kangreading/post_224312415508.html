PART 1 · Chapter 3

문장 맨 앞 Why절과 How절은 주어

강쌤의 포인트 │ 이 패턴에서 이렇게 생각하라

why와 how는 질문 표지가 아니라 이유·방법·정도를 품은 설명 덩어리로 움직인다. 먼저 던져야 할 질문은 이것이다:

"이건 질문인가, 설명하는 절인가?"

판별 순서:

1

문장 맨 앞의 why/how를 확인한다

2

why/how 뒤에 나오는 동사 2개를 확인한다

3

두 번째 동사 바로 앞에서 대괄호 [ ]를 닫는다

4

대괄호 전체를 주어로 본다

why와 how가 문장 맨 앞에 있을 때 의문문처럼 보인다. 하지만 이것은 이유나 방법·정도를 나타내는 주어 덩어리를 만든다.

▸ why + 주어 + 동사 ~ + 중심동사 → why절 전체가 주어

▸ how + 주어 + 동사 ~ + 중심동사 → how절 전체가 주어

▸ how + 형용사/부사 + 주어 + 동사 ~ + 중심동사 → how절 전체가 주어

참고 │ Whether절·Where절·When절과의 결정적 차이

Why절과 How절은 문장 맨 앞에서 부사어로 쓰이지 않는다. 따라서 콤마 유무를 확인할 필요가 없다. 문장 맨 앞에 나오면 무조건 절 전체를 주어로 묶으면 된다.

1. Why: 주어 절의 시작

강쌤의 포인트 │ Why

why는 "왜 그렇게 했는지" / "왜 그런 일이 일어났는지"처럼 이유 전체를 하나의 절로 품는다. why절 안에서 동사가 먼저 보여도 거기서 문장이 끝나는 것이 아니다. 진짜 중심동사는 그 뒤에 따로 나온다.

예문 1 Why절 → 주어

예문 1

[Why he resigned so suddenly] remains a mystery.

(그가 왜 그렇게 갑자기 사임했는지는 여전히 수수께끼다.)

  • Why he resigned so suddenly = 주어 (why절)

  • remains = 중심동사

  • a mystery = 보어

동사 resigned와 remains 중 둘째 동사 remains 앞에서 대괄호를 닫는다. 대괄호 전체가 주어이다. 주어가 '이유'나 '의문'을 나타내기 때문에, 동사 자리에 remains, 보어 자리에 a mystery나 unknown 같은 단어가 오는 것이 논리적으로 매우 자연스럽다.

🔗 꼬꼬무 논리 분석

  • 꼬꼬무 공식: [왜 그런 일이 벌어졌는가] (꼬리) → [그 사실이 어떠하다] (새로운 정보)

  • Why절을 하나의 거대한 '이유 덩어리'로 인식하게 되면, 뒤에 오는 보어 mystery는 그 상자의 겉면에 붙은 라벨처럼 명확하게 이해된다.

예문 2 Why절 → 주어 (부정 보어)

예문 2

[Why the project failed] is not our primary concern.

(그 프로젝트가 왜 실패했는지는 우리의 주된 관심사가 아니다.)

  • Why the project failed = 주어 (why절)

  • is = 중심동사

  • not = 부사어

  • our primary concern = 보어

동사 failed와 is 중 is 앞에서 대괄호를 닫는다. "프로젝트가 실패했다"라는 결과보다, "도대체 왜 실패했는가"라는 분석적 의문을 하나의 덩어리로 묶어 판단의 대상으로 삼고 있다.

🔗 꼬꼬무 논리 분석

  • 글쓴이의 전략: "원인 규명보다 더 중요한 게 따로 있다"는 것을 강조하기 위해, 사람들이 가장 궁금해할 만한 Why절을 문장 맨 앞에 미끼로 던진 뒤, not으로 과감하게 처리한 것이다.

  • 꼬꼬무 흐름: [왜 실패했는가? — 미끼] → [우리의 주된 관심사가 아니다] → 독자의 시선을 다음 문장으로 유도하는 논리적 전략

예문 3 That + Why절: 이중 구조

예문 3

[ That { (why he lied) is still a mystery } ] is true.

(왜 그가 거짓말했는지는 여전히 수수께끼라는 것은 사실이다.)

  • That why he lied is still a mystery = 주어 (that절)

  • why he lied = 첫 번째 is의 주어

  • is (세 번째) = 중심동사

  • true = 보어

That 뒤에 동사 3개 lied, is, is가 나오고 있다. 세 번째 동사 is가 중심동사이다.

🔗 꼬꼬무 구조 독해 (마디 시각화)

[ That { (why he lied) is still a mystery } ] is true.

  • 소괄호 ( ) : 거짓말 이유 (왜 거짓말했는가)

  • 중괄호 { } : 상태 (여전히 수수께끼다)

  • 대괄호 [ ] : 규정 (수수께끼라는 그 사실)

  • 논리 확장: [거짓말 이유] → [모름] → [모른다는 사실] → [진실임] 순으로 꼬리에 꼬리를 물며 논리가 완결된다.

2. How: 주어 절의 시작

강쌤의 포인트 │ How

how는 내용에 따라 두 방향으로 간다. 방법 절("어떻게 ~하는지")과 정도 절("얼마나 ~하는지")이다. 둘 다 절 전체가 하나의 주어가 된다는 점은 같다.

How + 주어 + 동사

→ 방법 절

"어떻게 ~하는지"

How + 형용사/부사 + 주어 + 동사

→ 정도 절

"얼마나 ~하는지"

1) 방법을 나타내는 How

강쌤의 포인트 │ How (방법)

How 바로 뒤에 주어가 나오면 how는 "어떻게 ~하는지"라는 방법 절의 시작을 나타낸다.

예문 1 방법 How → 주어

예문 1

[How he solved the problem] surprised everyone.

(어떻게 그가 그 문제를 풀었는지는 모두를 놀라게 했다.)

  • How he solved the problem = 주어 (방법을 나타내는 how절)

  • surprised = 중심동사

  • everyone = 목적어

동사 solved와 surprised 중 둘째 동사 surprised 앞에서 대괄호를 닫는다. How 바로 뒤에 주어 he가 나오고 있으므로 how는 '어떻게 ~하는지'라는 방법을 나타낸다.

🔗 꼬꼬무 논리 분석

  • 꼬꼬무 공식: [어떤 기막힌 방식으로 해결했는가] (꼬리) → [그 사실이 충격을 주었다] (새로운 정보)

  • How절을 하나의 거대한 '방법 덩어리'로 인식하게 되면, 뒤에 오는 동사는 그 덩어리가 세상에 끼친 영향력을 설명하는 것으로 자연스럽게 읽힌다.

예문 2 방법 How → 주어

예문 2

[How you sleep] determines your health.

(어떻게 당신이 잠을 자느냐가 당신의 건강을 결정한다.)

  • How you sleep = 주어 (방법을 나타내는 how절)

  • determines = 중심동사

  • your health = 목적어

동사 sleep과 determines 중 둘째 동사 determines 앞에서 대괄호를 닫는다. "당신은 잠을 잔다"라는 단순한 사실이 아니라, "어떤 방식으로(How) 잠을 자는가"라는 질적인 과정이 건강의 결정 요인임을 논리적으로 서술하고 있다.

🔗 꼬꼬무 논리 분석

  • 정보의 확장: [잠자는 방식 (꼬리)] → [결정한다 (연결)] → [당신의 건강을 (새로운 정보)] 순으로 정보가 꼬리에 꼬리를 물며 확장된다.

예문 3 That + How절 (방법): 이중 구조

예문 3

[ That { (how you study) matters most } ] is my point.

(어떻게 당신이 공부하느냐가 가장 중요하다는 것이 나의 핵심 주장이다.)

  • That how you study matters most = 주어 (that절)

  • (how you study) = matters의 주어

  • is = 중심동사

  • my point = 보어

That 뒤에 동사 3개 study, matters, is가 나오고 있다. 세 번째 동사 is가 중심동사이다.

🔗 꼬꼬무 구조 독해 (마디 시각화)

[ That { (how you study) matters most } ] is my point.

  • 소괄호 ( ) : 공부하는 방식 (어떻게 공부하는가)

  • 중괄호 { } : 그 방식이 가장 중요하다

  • 대괄호 [ ] : 그 방식이 가장 중요하다는 사실

  • '어떻게 공부하는가(how ~)'라는 방법이 '가장 중요하다(matters most)'는 결론을 하나의 객관적 사실(That ~)로 묶어, 그것이 바로 내가 말하고자 하는 핵심(my point)임을 선언하고 있다.

2) 정도를 나타내는 How

강쌤의 포인트 │ How (정도)

How 바로 뒤에 형용사나 부사가 나오면 how는 "얼마나 ~하는지"라는 정도를 나타낸다. 문장 맨 앞의 How절은 명사절로서 문장 전체의 주어로 쓰인다.

예문 1 정도 How (many) → 주어

예문 1

[How many languages you speak] determines your career opportunities.

(네가 얼마나 많은 언어를 말하느냐가 너의 경력 기회들을 결정한다.)

  • How many languages you speak = 주어 (정도를 나타내는 how절)

  • determines = 중심동사

  • your career opportunities = 목적어

동사 speak와 determines 중 둘째 동사 determines 앞에서 대괄호를 닫는다. How 바로 뒤에 형용사 many가 나오고 있으므로 how는 '얼마나 ~하는지'라는 정도를 나타낸다.

🔗 꼬꼬무 구조 독해

[ (How many languages) you speak ] → ( determines ) → [ your career opportunities ].

  • 핵심 Point: How는 형용사/부사와 결합해서 하나의 '정도 덩어리'를 만든다.

  • 독해 Tip: How 뒤에 형용사/부사가 보이면, "얼마나 ~한지"라고 해석하라. How가 이끄는 명사절은 둘째 동사 바로 앞까지 이어지고, 문장 전체의 주어가 된다.

​

예문 2 정도 How (much) → 주어

예문 2

[How much we can achieve] depends on our cooperation.

(우리가 얼마나 많이 성취할 수 있는지는 우리의 협력에 달려 있다.)

  • How much we can achieve = 주어 (정도를 나타내는 how절)

  • depends = 중심동사

  • on our cooperation = 부사어

동사 achieve와 depends 중 둘째 동사 depends 앞에서 대괄호를 닫는다. How 바로 뒤에 부사 much가 나오고 있으므로 how는 '얼마나 ~하는지'라는 정도를 나타낸다.

🔗 꼬꼬무 구조 독해

[ (How much) we can achieve ] → ( depends on ) → [ our cooperation ].

  • 핵심 Point: How는 뒤의 much와 한 몸이 되어 '얼마나 많이'라는 거대한 측정 장치를 만든다.

  • 독해 Tip: depends라는 중심동사를 만나는 순간, "앞의 '성취의 양'은 아직 미정이고, 뒤의 '협력'이 그 값을 정해주겠구나!"라고 예측하며 읽어야 한다.

주의 │ 방법 How vs 정도 How 빠른 구분법

How + 주어 + 동사

How + 형용사/부사 + 주어 + 동사

→ 방법절: "어떻게 ~하는지"

예: How he solved it

→ 정도절: "얼마나 ~하는지"

예: How many / How much

Chapter 3 정리 │ Why절과 How절 주어 판별 기준

  • Why절과 How절 공통: 문장 맨 앞에서 항상 주어이다. 콤마 유무 확인 불필요

  • 이유 절: Why + 주어 + 동사 ~ → "왜 ~하는지"

  • 방법 절: How + 주어 + 동사 ~ → "어떻게 ~하는지"

  • 정도 절: How + 형용사/부사 + 주어 + 동사 ~ → "얼마나 ~하는지"

  • That + why/how절: 세 번째 동사가 중심동사 — 내부 절은 소괄호로 묶는다

  • Chapter 2와의 차이: Whether/Where/When절은 콤마 유무로 주어↔부사어가 갈리지만, Why/How절은 항상 주어
