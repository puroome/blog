PART 1 · Chapter 2

문장 맨 앞 Whether절, Where절, When절은

주어이거나 부사어

강쌤의 포인트 │ 이 패턴에서 이렇게 생각하라

Whether절, Where절, When절을 보면 뜻보다 구조를 먼저 따져라. 먼저 던져야 할 질문은 이것이다:

"이 절이 주어인가, 그 뒤의 절을 수식하는 부사어인가?"

판별 기준: 콤마가 있는지 보라

콤마 없음 → 주어

두 번째 동사 앞에서 [ ]를 닫는다

→ 대괄호 전체 = 주어

콤마 있음 → 부사어

콤마 앞에서 ( )를 닫는다

→ 소괄호 전체 = 부사어

콤마 없을 때 절 전체의 역할을 파악하는 순서:

1

whether/where/when 뒤에 콤마가 없다

2

뒤에 나오는 동사 2개를 확인한다

3

두 번째 동사 바로 앞까지 whether/where/when절이다

4

두 번째 동사 앞에서 대괄호 [ ]를 닫는다

5

대괄호 전체를 주어로 본다

콤마 있을 때 절 전체의 역할을 파악하는 순서:

1

whether/where/when 뒤에 콤마가 있다

2

콤마 앞에서 소괄호 ( )를 닫는다

3

소괄호 전체를 부사어로 본다

참고 │ Why절/How절과의 차이 (Chapter 3)

Whether절·Where절·When절은 주어와 부사어 둘 다 가능하다. 반면 Why절과 How절은 문장 맨 앞에서 항상 주어로만 쓰인다. 콤마 유무를 확인할 필요가 없다.

1. Whether: 주어 절의 시작인가, 부사어 절의 시작인가

강쌤의 포인트 │ Whether

whether는 "올까, 안 올까" / "이길까, 질까"처럼 둘 중 어느 쪽인가라는 갈림길을 나타내는 절을 품는다. 그 갈림길 절이 문장 안에서 무슨 역할을 하는지가 핵심이다.

콤마 없음

whether절 전체가 주어

"~인지 아닌지(가)"

콤마 있음

whether절 전체가 부사어

"~이든지 아니든지"

예문 1 콤마 없음 → Whether절 = 주어

예문 1

[Whether the new policy will work] remains uncertain.

(그 새로운 정책이 효과가 있을지는 불확실한 채로 남아 있다.)

  • Whether the new policy will work = 주어 (whether절)

  • remains = 중심동사

  • uncertain = 보어

콤마 없다. 동사 will work과 remains 중 두 번째 remains 앞에서 대괄호를 닫는다. 대괄호 전체가 주어이다.

이 문장은 "그 정책이 효과가 있을 것인가, 없을 것인가"라는 의문 자체를 주어 자리에 앉힌 구조이다. 보어 자리에 uncertain이 오는 이유는, 주어인 whether절 자체가 이미 "모른다"는 의미를 내포하기 때문이다.

🔗 꼬꼬무 논리 분석

  • 꼬꼬무 공식: [효과가 있을지(A) 혹은 없을지(B)] (꼬리) → [그 상태가 유지된다] (연결) → [불확실하게] (결과)

  • 콤마가 없는 긴 주어 절에서 두 번째 동사 remains 앞을 딱 끊어내는 감각! 이것이 바로 구조가 먼저 보이는 '고득점 독해'의 지름길이다.

예문 2 콤마 있음 → Whether절 = 부사어

예문 2

(Whether the new policy will work), critics are already preparing objections.

(그 새로운 정책이 효과가 있든지 없든지, 비판자들은 이미 반대를 준비하고 있다.)

  • Whether the new policy will work = 부사어 (갈림길 배경)

  • critics = 주어

  • are preparing = 중심동사

  • objections = 목적어

콤마가 있다. 콤마 앞에서 소괄호를 닫는다. 소괄호 전체가 부사어이다. "정책의 성공 여부와 상관없이"라는 배경 상황을 먼저 던지고, 그 뒤 주절에서 진짜 하고 싶은 말을 이어가는 구조이다.

🔗 꼬꼬무 논리 분석

  • 글쓴이의 전략: "정책의 성패"라는 변수를 부사어 자리로 밀어내 버림으로써, 비판자들의 반대는 이미 정해진 수순이라는 점을 논리적으로 강조하려는 의도이다.

  • 꼬꼬무 흐름: [(정책 성공 여부는) 배경으로 처리] → [비판자들은 이미 반대를 준비 중이다] → 논리적 확정

예문 3 That + Whether절: 이중 구조

예문 3

[ That { (whether he survived) is unknown } ] is worrisome.

(그가 생존했는지 여부가 알려지지 않았다는 것은 걱정스럽다.)

  • That whether he survived is unknown = 주어 (that절)

  • whether he survived = 첫 번째 is의 주어

  • is (두 번째) = 중심동사

  • worrisome = 보어

동사 3개: survived, is, is — 마지막 is 앞에서 대괄호를 닫는다. 대괄호 전체가 주어다.

🔗 꼬꼬무 구조 독해 (마트료시카 시각화)

[ That { (whether he survived) is unknown } ] is worrisome.

  • 소괄호 ( ) : 의문의 시작 (생존했나?)

  • 중괄호 { } : 상황 (모르는 상태)

  • 대괄호 [ ] : 상황의 규정 (모르는 상태라는 사실)

  • 논리 체계: [질문(whether)] → [상태(is unknown)] → [사실(That)] → [감정/판단(is worrisome)]

2. Where: 주어 절의 시작인가, 부사어 절의 시작인가

강쌤의 포인트 │ Where

where는 "어디에서 발생하는가" / "어디가 그 지점인가"처럼 장소 정보를 하나의 절로 품는다. 그 장소 절이 문장 안에서 무슨 역할을 하는지가 핵심이다.

콤마 없음

where절 전체가 주어

"~하는 곳은"

콤마 있음

where절 전체가 부사어

"~하는 곳에서"

예문 1 콤마 없음 → Where절 = 주어

예문 1

[Where the two roads meet] is marked by an old stone tower.

(두 도로가 만나는 곳은 오래된 돌탑으로 표시된다.)

  • Where the two roads meet = 주어 (where절)

  • is marked = 중심동사

  • by an old stone tower = 부사어

콤마가 없다. 동사 meet와 is marked 중 is marked 앞에서 대괄호를 닫는다. 대괄호 전체가 주어이다. where절은 사실상 The place where ~에서 선행사인 The place가 생략된 형태로 볼 수 있다.

🔗 꼬꼬무 논리 분석

  • 꼬꼬무 공식: [어디에서 무엇이 일어나는지] (꼬리) → [그 장소가 어떠하다] (새로운 정보)

  • meet이라는 동사를 보고 문장이 끝났다고 착각하지 않는 것이 핵심. Where절을 하나의 거대한 '장소 덩어리'로 인식하면, 뒤의 is marked를 찾는 속도가 비약적으로 빨라진다.

예문 2 콤마 있음 → Where절 = 부사어

예문 2

(Where the two roads meet), travelers often stop to rest.

(두 도로가 만나는 곳에서, 여행자들은 자주 멈춰서 휴식을 취한다.)

  • Where the two roads meet = 부사어 (장소 배경)

  • travelers = 주어

  • often = 부사어

  • stop = 중심동사

  • to rest = 부사어

콤마가 있다. 콤마 앞에서 소괄호를 닫는다. 소괄호 전체가 부사어이다. 콤마는 "Where절은 배경일 뿐이니, 진짜 중요한 내용은 뒤에 있다!"고 독자에게 알려주는 신호이다.

🔗 꼬꼬무 논리 분석

  • 글쓴이의 전략: "두 길이 만나는 곳"이라는 특수한 장소가 주는 필연성을 먼저 강조함으로써, 뒤에 오는 '휴식'이라는 결과에 타당한 근거를 부여하려는 설계이다.

  • 꼬꼬무 흐름: [(두 도로가 만나는 곳이라는) 장소 배경] → [여행자들이 자주 멈춘다] → 장소의 필연성 강조

예문 3 That + Where절: 이중 구조

예문 3

[ That { (where they stay) remains secret } ] is understandable.

(그들이 어디에 머무는지가 비밀로 유지된다는 것은 이해할 만하다.)

  • That where they stay remains secret = 주어 (that절)

  • where they stay = remains의 주어

  • is = 중심동사

  • understandable = 보어

동사 3개: stay, remains, is — 마지막 is 앞에서 대괄호를 닫는다. 대괄호 전체가 주어이다.

🔗 꼬꼬무 구조 독해 (정보의 겹포장)

[ That { (where they stay) remains secret } ] is understandable.

  • 소괄호 (속포장) : 어디에 있는지(where)

  • 중괄호 {중간포장} : 비밀로 남은 상태(remains secret)

  • 대괄호 [겉포장] : 비밀로 남아있다는 사실(That) → 이해할 수 있다

  • "마지막 동사가 중심동사다"라는 원칙이 이 같은 복잡한 문장을 분석하는 핵심 무기다.

3. When: 주어 절의 시작인가, 부사어 절의 시작인가

강쌤의 포인트 │ When

when은 "언제 일어나는가" / "어느 시점인가"처럼 시간 정보를 하나의 절로 품는다. 그 시간 절이 문장 안에서 무슨 역할을 하는지가 핵심이다.

콤마 없음

when절 전체가 주어

"~하는 때는"

콤마 있음

when절 전체가 부사어

"~할 때"

예문 1 콤마 없음 → When절 = 주어

예문 1

[When the negotiations began] remains unclear.

(그 협상이 시작된 때는 불분명한 채로 남아 있다.)

  • When the negotiations began = 주어 (when절)

  • remains = 중심동사

  • unclear = 보어

콤마가 없다. 동사 began과 remains 중 remains 앞에서 대괄호를 닫는다. 대괄호 전체가 주어이다. 문장 중간에 콤마가 없다는 것은 When절이 문장의 필수 성분(주어)임을 뜻한다.

🔗 꼬꼬무 논리 분석

  • 꼬꼬무 공식: [언제 일이 시작되었는가] (꼬리) → [그 사실이 어떠하다] (새로운 정보)

  • When절을 하나의 거대한 '시간 덩어리'로 인식하면, 뒤에 오는 보어 unclear를 주어의 당연한 결과값으로 자연스럽게 읽을 수 있다.

예문 2 콤마 있음 → When절 = 부사어

예문 2

(When the negotiations began), few observers expected success.

(그 협상이 시작되었을 때, 관찰자들은 거의 성공을 기대하지 않았다.)

  • When the negotiations began = 부사어 (시간 배경)

  • few observers = 주어

  • expected = 중심동사

  • success = 목적어

콤마가 있다. 콤마 앞에서 소괄호를 닫는다. 소괄호 전체가 부사어이다. "어떤 시점에(배경)" + "누가 무엇을 했다(본론)"라는 논리적 흐름을 가지고 있다.

🔗 꼬꼬무 논리 분석

  • 글쓴이의 전략: 협상이 시작되었다는 사실보다, '시작 당시의 차가운 여론'을 강조하고 싶어한다. "협상이 시작되었다"라는 배경을 먼저 깔아 독자가 기대감을 갖게 만든 뒤, 곧바로 few라는 부정 의미를 던져 그 기대감을 반전시키는 논리적 충격을 주려는 의도이다.

  • 꼬꼬무 흐름: [(협상이 시작된) 시간 배경] → [성공 기대 없음] → 역설적 분위기 강조

예문 3 That + When절: 이중 구조

예문 3

[ That { (when she arrives) depends on traffic } ] is clear.

(그녀가 언제 도착하는지는 교통 상황에 달려 있다는 것은 명확하다.)

  • That when she arrives depends on traffic = 주어 (that절)

  • (when she arrives) = depends의 주어

  • is = 중심동사

  • clear = 보어

동사 3개: arrives, depends, is — 마지막 is 앞에서 대괄호를 닫는다. 대괄호 전체가 주어이다.

🔗 꼬꼬무 구조 독해 (마디 시각화)

[ That { (when she arrives) depends on traffic } ] is clear.

  • 소괄호 ( ) : 의문의 시점 (언제 도착하는가)

  • 중괄호 { } : 상황 (교통 상황에 달려있다)

  • 대괄호 [ ] : 상황의 규정 (교통 상황에 달려있다는 사실)

  • 세 번째 동사를 찾아 마디를 닫는 훈련은 고난도 지문에서 가장 강력한 무기가 된다.

Chapter 2 정리 │ Whether절, Where절, When절 판별 기준

  • 콤마 없음: 두 번째 동사 앞에서 대괄호 [ ]를 닫는다 → 절 전체가 주어

  • 콤마 있음: 콤마 앞에서 소괄호 ( )를 닫는다 → 절 전체가 부사어

  • That + whether/where/when: 세 번째 동사가 중심동사 — 내부 절은 소괄호로 묶는다

  • Why절·How절과의 차이: whether/where/when절은 주어와 부사어 둘 다 가능; why절과 how절은 문장 맨 앞에서 항상 주어

  • 검증 공식: 동사 개수 − 1 = 접속사/관계사 개수
