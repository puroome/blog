PART 1 · Chapter 4

문장 맨 앞 'To + 동사원형'과 '-ing'는

주어이거나 부사어

강쌤의 포인트 │ 이 패턴에서 이렇게 생각하라

문장 맨 앞에 to부정사나 -ing가 나오면, 형태 이름보다 먼저 역할을 파악해야 한다. 주어인지 부사어인지 구분하는 기준은 단 하나다:

"콤마가 있는가, 없는가?"

콤마 없음 → 주어

두 번째 동사 앞에서 [ ]를 닫는다

→ 대괄호 전체 = 주어

"~하는 것은"

콤마 있음 → 부사어

콤마 앞에서 ( )를 닫는다

→ 소괄호 전체 = 부사어

"~하기 위해 / ~하면서"

판별 순서 (콤마 없을 때):

1

to부정사/-ing 뒤에 콤마가 없다

2

뒤에 나오는 동사 2개를 확인한다

3

두 번째 동사 바로 앞에서 대괄호 [ ]를 닫는다

4

대괄호 전체를 주어로 본다

판별 순서 (콤마 있을 때):

1

to부정사/-ing 뒤에 콤마가 있다

2

콤마 앞에서 소괄호 ( )를 닫는다

3

소괄호 뒤 진짜 주어와 중심동사를 찾는다

참고 │ Chapter 2와의 공통점

Whether/Where/When절과 동일하게 콤마 유무로 주어와 부사어가 갈린다. 이 판별 공식은 PART 1 전체를 관통하는 핵심 원칙이다.

1. To부정사: 주어인가, 부사어인가

강쌤의 포인트 │ To부정사

문장 맨 앞의 to부정사는 목적을 나타내는 부사어로 쓰이는 경우가 많아 학생들이 습관적으로 '~하기 위해'로 먼저 읽는다. 하지만 콤마 없이 중심동사가 바로 이어지면 주어다.

주의 │ 이 함정에 빠지지 마라

문장 맨 앞의 to부정사를 보자마자 '~하기 위해'로 단정하면 안 된다. 콤마가 없고 뒤에 중심동사가 이어지면, to부정사는 '~하는 것' 의미의 주어이다.

예문 1 콤마 없음 → To부정사 = 주어

예문 1

[To master a new language] requires constant practice.

(새로운 언어를 익히는 것은 끊임없는 연습을 필요로 한다.)

  • To master a new language = 주어 (to부정사구)

  • requires = 중심동사

  • constant practice = 목적어

to부정사 뒤에 콤마가 없다. 중심동사 requires 앞에서 대괄호를 닫는다. 대괄호 전체가 주어이다. language 뒤에 콤마가 없다는 것이 to부정사구가 주어라는 확실한 신호이다.

참고 │ To부정사 주어의 기원

[To master a new language] = [That you master a new language]

That절이 문장 맨 앞에 오면 무겁게 느껴지기 때문에, 실제 문장에서는 to부정사나 동명사를 선호한다.

🔗 꼬꼬무 논리 분석

  • 정보의 확장: [언어 정복이라는 목표 (꼬리)] → [요구한다 (연결)] → [끊임없는 연습을 (새로운 정보)] 순으로 정보가 꼬리에 꼬리를 물며 확장된다.

  • '언어'라는 대상보다 '언어를 익히는 행위' 자체를 주어 자리에 놓아, 그 행위에 필요한 필수 조건을 서술하고 있다.

예문 2 콤마 있음 → To부정사 = 부사어

예문 2

(To master a new language), you must practice every day.

(새로운 언어를 익히기 위해서, 너는 매일 연습해야 한다.)

  • To master a new language = 부사어 (목적)

  • you = 주어

  • must practice = 중심동사

  • every day = 부사어

to부정사 뒤에 콤마가 있다. 콤마 앞에서 소괄호를 닫는다. 소괄호 전체가 부사어이다. 콤마는 "여기까지는 부차적인 설명(부사어)이었고, 이제부터 진짜 주어와 동사가 시작된다"라고 알려주는 신호이다.

참고 │ To부정사 부사어의 기원

(To master a new language) = (In order that you can master a new language)

'To master ~'는 일상적으로 널리 쓰이는 반면, 'In order that~'은 격식 있는 문어체에서 주로 쓰인다.

🔗 꼬꼬무 논리 분석

  • 글쓴이의 전략: "너는 매일 연습해야 해"라고 불쑥 말하는 대신, [언어 정복이라는 매력적인 목표]를 먼저 독자의 눈앞에 보여준다. "네가 원하는 게 이거지? 그렇다면..."이라며 논리적 명분을 먼저 세운 뒤, 그 뒤에 '매일 연습'이라는 다소 힘든 조건을 붙임으로써 독자가 그 제안을 거부할 수 없게 만드는 설득의 논리를 구사한 것이다.

예문 3 콤마 유무 비교 (S1 vs S2)

예문 3

S1 — 콤마 없음 → 주어

[To revise the manuscript] requires patience.

(원고를 수정하는 것은 인내를 요구한다.)

  • To revise the manuscript = 주어

  • requires = 중심동사 → requires 앞에서 [ ] 닫음

S2 — 콤마 있음 → 부사어

(To avoid unnecessary delay), we changed our schedule.

(불필요한 지연을 피하기 위해서, 우리는 일정을 변경했다.)

  • To avoid unnecessary delay = 부사어

  • we = 주어 / changed = 중심동사 → 콤마 앞에서 ( ) 닫음

🔗 꼬꼬무 구조 독해 (S1 vs S2 비교)

  • S1의 초점 (행위의 성격): "원고 수정"이라는 행위 자체가 문장의 핵심 정보다. "이 일은 인내가 필요한 작업이야"라는 행위의 속성을 강조한다.

  • S2의 초점 (행위의 목적): "우리가 일정을 바꾼 것"이 핵심이고, "지연 방지"는 그 이유를 설명하는 보조 정보다. 콤마가 정보의 계급을 정한다.

2. -ing: 주어인가, 부사어인가

강쌤의 포인트 │ -ing

문장 맨 앞의 -ing는 두 가지로 나뉜다. 콤마 없이 중심동사가 바로 이어지면 동명사구로서 주어이고, 콤마 뒤에 새 주어가 나오면 분사구문으로서 부사어다.

콤마 없음 → 동명사구 (주어)

"~하는 것은"

→ 대괄호로 묶어 주어로 확정

콤마 있음 → 분사구문 (부사어)

"~하면서 / ~한 후에"

→ 소괄호로 묶어 부사어로 확정

예문 1 콤마 없음 → -ing = 주어 (동명사구)

예문 1

[Reading historical documents] takes time.

(역사 문서들을 읽는 것은 시간을 필요로 한다.)

  • Reading historical documents = 주어 (동명사구)

  • takes = 중심동사

  • time = 목적어

-ing 뒤에 콤마가 없다. 중심동사 takes 앞에서 대괄호를 닫는다. 대괄호 전체가 주어이다. -ing 안의 동사 reading에 끌려 문장이 이미 시작됐다고 착각하지 않는 것이 핵심이다.

🔗 꼬꼬무 논리 분석

  • 꼬꼬무 공식: [역사 문서를 읽는다는 행위 (꼬리)] → [요구한다 (연결)] → [시간을 (새로운 정보)]

  • 동명사구를 하나의 거대한 '행위 덩어리'로 인식하면, takes라는 중심동사가 선명하게 보인다.

예문 2 콤마 있음 → -ing = 부사어 (분사구문)

예문 2

(Reviewing the final draft), the editor noticed several inconsistencies.

(최종 초안을 검토하면서, 그 편집자는 몇 가지 불일치를 발견했다.)

  • Reviewing the final draft = 부사어 (분사구문 — 동시 동작)

  • the editor = 주어

  • noticed = 중심동사

  • several inconsistencies = 목적어

-ing 뒤에 콤마가 있다. 콤마 앞에서 소괄호를 닫는다. 콤마는 "여기까지는 상황 설명일 뿐이야"라고 선언하며 진짜 주어(the editor)에게 자리를 내어준다.

🔗 꼬꼬무 논리 분석

  • 글쓴이의 전략: "편집자가 불일치를 발견했다"는 것이 핵심이고, "초안 검토"는 그저 배경이다. 진짜 하고 싶은 말은 편집자가 찾아낸 결과물이다.

  • 꼬꼬무 흐름: [(초안을 검토하는) 배경 설정] → [편집자가 발견했다] → 행동의 결과 강조

예문 3 콤마 유무 비교 (S1 vs S2) — 핵심 정리

예문 3

S1 — 콤마 없음 → 주어 (동명사구)

[Reading historical documents] takes time.

(역사 문서들을 읽는 것은 시간을 필요로 한다.)

S2 — 콤마 있음 → 부사어 (분사구문)

(Reviewing the final draft), the editor noticed several inconsistencies.

(최종 초안을 검토하면서, 그 편집자는 몇 가지 불일치를 발견했다.)

🔗 꼬꼬무 구조 독해 (S1 vs S2 비교)

  • S1의 초점 (행위의 속성): '역사 문서를 읽는다는 것'이 어떤 성질을 가졌는지 말하고 싶어한다. "이 일은 시간이 꽤 걸리는 작업이야"라는 행위의 속성이 핵심 메시지다.

  • S2의 초점 (주인공의 행적): "편집자"가 무엇을 했는지에 집중한다. "초안 검토"는 배경일 뿐, 진짜 하고 싶은 말은 편집자가 '불일치를 찾아냈다'는 결과물이다.

  • 결론: 콤마가 없으면 -ing가 주인공(주어)이고, 콤마가 있으면 -ing는 배경(부사어)이다. 콤마 하나가 정보의 계급을 정한다.

Chapter 4 정리 │ To부정사/-ing 판별 공식

  • 판별 기준: 콤마가 있는지 먼저 확인하라

  • 콤마 없음 → 주어: [To부정사 / -ing] + 중심동사 → 앞부분 전체가 주어 ("~하는 것은")

  • 콤마 있음 → 부사어: (To부정사 / -ing), + 주어 + 중심동사 → 콤마 앞 전체가 부사어 ("~하기 위해 / ~하면서")

  • To부정사 주어의 기원: That절의 축약 형태 → [To V~] = [That S V~]

  • To부정사 부사어의 기원: In order that절의 축약 형태 → (To V~) = (In order that S V~)

  • Chapter 2와의 공통점: Whether/Where/When절과 동일하게 콤마 유무로 주어↔부사어가 결정된다

  • Chapter 3과의 차이: Why절·How절은 콤마 유무와 무관하게 항상 주어; To부정사/-ing는 콤마 유무로 갈린다
